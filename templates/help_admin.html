시스템

운영 가이드

현재 적용된 목요일 본 세미나·월요일 추가 세미나 운영 방식을 기준으로 정리했습니다.

개요세미나 운영발제문조 편성·기록회원 관리게시판·위키시스템 점검

개요

운영 데이터의 기준은 세미나 운영 화면입니다. 도서와 발제문은 목요일 본 세미나부터 다음 월요일 추가 세미나까지 한 묶음으로 관리하고, 참석자와 조 편성 결과는 회차별로 따로 저장합니다.

권장 순서 · 도서 입력 → 발제문 수집 열기 → 목요일 불참 입력 또는 월요일 신청 확인 → 조 편성 → 진행 기록 확인

세미나 운영

1
도서 일정

카드에 목요일 본 세미나와 그다음 월요일 추가 세미나 날짜를 함께 표시합니다. 예: 7/23(목) 본 세미나 → 7/27(월) 추가 세미나.

2
목요일 본 세미나

회원 투표를 받지 않습니다. 카카오톡으로 받은 불참자를 운영진이 등록하면 조 편성에서 자동 제외됩니다.

3
월요일 추가 세미나

회원이 학번과 이름을 확인한 뒤 신청합니다. 기본 정원은 10명입니다. 운영진이 숫자를 바꾼 뒤 이 정원으로 변경을 눌러야 새 정원이 반영되고, 신청 취소 시 자리가 다시 열립니다.

회차나 학기를 없애야 할 때는 기록 보존을 위해 삭제보다 보관·비활성화를 우선하세요.

발제문

1
세미나 운영 카드에서 발제문 수집 열기를 한 번 누릅니다.
2
회원은 기본 1개를 제출하고, 회장은 필요하면 추가 발제를 작성할 수 있습니다.
3
제출 화면의 먼저 나온 발제문 보기에서 이름·학과·학번 없이 기존 내용을 확인해 중복을 피할 수 있습니다.
4
제출 관리에서 확인한 뒤 Word를 내려받습니다. Word에는 작성자별 학과·학번·이름이 포함됩니다.

조 편성·기록

1
세미나 운영 카드의 해당 회차에서 조 편성을 엽니다.
2
목요일은 활성 회원에서 불참자를 제외하고, 월요일은 신청 확정자만 자동 선택됩니다.
3
발제문 제출자는 학번을 우선으로 발제자에 연결합니다. 자동 연결 실패 목록은 현장에서 확인합니다.
4
결과를 저장하면 회차 날짜로 진행 기록과 만남 매트릭스가 갱신됩니다. 과거 조를 고치면 전체 이력 기준으로 다시 계산됩니다.

회원 관리

운영 관리 → 회원 관리에서 검색, 정보 편집, 활성·휴면·비활성 전환과 중복 계정 합치기를 처리합니다.

  • 활성: 목요일 예상 참석자와 조 편성 후보에 포함
  • 휴면: 기록은 유지하지만 기본 운영 명단에서 제외
  • 비활성: 졸업·탈퇴 등 장기 미활동 회원
  • 이메일은 선택 사항이고, 학번·이름·학과는 본인 확인과 발제자 연결에 중요

게시판·위키

게시판 관리에서 공지·자유게시판·세미나 후기·벽돌책 후기의 이름, 권한, 댓글과 노출 순서를 관리합니다.

공식 안내 문서는 Notion 공개 위키를 사용합니다. 운영진 회의록과 회계 자료는 공개 위키 아래에 두지 말고 운영진 전용 영역에 보관하세요.

시스템 점검

  • 배포 후 세미나 운영 화면과 공개 발제문 링크를 각각 확인
  • Supabase에는 DB와 첨부파일을 저장하고 Render 디스크에는 영구 파일을 두지 않음
  • Render Starter 사용 시 별도 keep-alive 대신 health check만 유지
  • 오류 발생 시 회차를 다시 만들기 전에 연결된 발제문·진행 기록 여부를 먼저 확인
